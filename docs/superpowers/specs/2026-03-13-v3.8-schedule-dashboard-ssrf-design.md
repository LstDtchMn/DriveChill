**"Now" indicator:**

**Mobile layout:**

- No drag-to-create; tap empty cell opens popover

**Integration with Fan Curves page:**

## Phase 2: Dashboard Drag-and-Drop

### Context

### New Hook: `useDragReorder.ts`

**Location:** `frontend/src/hooks/useDragReorder.ts`

**API:**
```typescript
function useDragReorder<T>(
  items: T[],
  onReorder: (newItems: T[]) => void,
): {
  dragHandleProps: (index: number) => {
    onPointerDown: (e: React.PointerEvent) => void;
    style: React.CSSProperties;
    'aria-label': string;
    role: string;
    tabIndex: number;
    onKeyDown: (e: React.KeyboardEvent) => void;
  };
  itemProps: (index: number) => {
    style: React.CSSProperties;
    'data-drag-index': number;
  };
  dragIndex: number | null;
  dropIndex: number | null;
}
```

**Implementation (~80 lines):**
- `pointerdown` on drag handle: capture pointer, record start Y, set `dragIndex`
- `pointermove`: calculate which item index the pointer is over (by element positions), set `dropIndex`
- `pointerup`: reorder array, call `onReorder`, clear state
- Keyboard: Space to grab, ArrowUp/Down to move, Enter to drop, Escape to cancel
- Mobile: pointer events cover touch natively; long-press (300ms delay) to activate drag to avoid scroll conflict

**Visual feedback:**
- Dragged item: elevated with `box-shadow`, `scale(1.02)`, accent border
- Drop target: dashed border placeholder at insertion point
- Other items: smooth `transform: translateY()` transition to make room

### Changes to `SystemOverview.tsx`

- Remove ChevronUp/ChevronDown buttons from `CustomizePanel`
- Add drag handle (`⠿` grip icon) before each widget row
- Wire `useDragReorder` hook to the widget config array
- On reorder callback: renumber `order` values (e.g., `newItems.map((w, i) => ({ ...w, order: i }))`) to match the existing `loadWidgetConfig()` sort-by-order contract, then persist to localStorage
- Add ARIA attributes: `aria-grabbed`, `aria-dropeffect="move"`, and live-region announcements for screen readers (e.g., "Widget moved to position 3 of 6")

## Phase 3: Python DNS Rebinding SSRF Fix

### Context

The C# backend already pins resolved IPs via `SocketsHttpHandler.ConnectCallback` — it resolves DNS once, then forces the TCP connection to that IP, preventing DNS rebinding between resolution and connection.

The Python backend uses a plain `httpx.AsyncClient` which is vulnerable: an attacker controlling a machine's `base_url` DNS could return a safe IP during validation, then rebind to an internal IP (e.g., 169.254.169.254) before the actual HTTP request connects.

### Solution: Custom `httpx` Transport with IP Pinning

**Approach:** Create a custom `httpx.AsyncHTTPTransport` that resolves DNS before connecting and forces the connection to the resolved IP.

**Implementation:**

```python
class IPPinningTransport(httpx.AsyncHTTPTransport):
    """Resolves DNS once and pins the TCP connection to that IP."""

    def __init__(self, *, allow_private: bool = False, **kwargs):
        super().__init__(**kwargs)
        self._allow_private = allow_private

    async def handle_async_request(self, request: httpx.Request) -> httpx.Response:
        host = request.url.host
        # Resolve DNS
        infos = await asyncio.get_running_loop().getaddrinfo(
            host, request.url.port or (443 if request.url.scheme == "https" else 80),
            family=socket.AF_UNSPEC, type=socket.SOCK_STREAM,
        )
        if not infos:
            raise httpx.ConnectError(f"Cannot resolve host: {host}")

        ip = infos[0][4][0]

        # Validate resolved IP
        addr = ipaddress.ip_address(ip)
        if not self._allow_private and addr.is_private:
            raise httpx.ConnectError(f"Resolved to private IP — blocked")

        # Pin: replace hostname with resolved IP in the URL, set Host header
        pinned_url = request.url.copy_with(host=ip)
        merged_headers = dict(request.headers)
        merged_headers["Host"] = host  # preserve original Host for TLS SNI + vhosts
        request = httpx.Request(
            method=request.method,
            url=pinned_url,
            headers=merged_headers,
            content=request.content,
        )
        return await super().handle_async_request(request)
```

### Integration

In `machine_monitor_service.py`, replace:
```python
self._client = httpx.AsyncClient(follow_redirects=False)
```
with:
```python
self._client = httpx.AsyncClient(
    transport=IPPinningTransport(allow_private=settings.allow_private_outbound_targets),
    follow_redirects=False,
)
```

### Validation Changes

The existing `validate_outbound_url_at_request_time()` function validates the URL *before* the request but doesn't pin the IP. With the transport-level pinning, the validation is now defense-in-depth rather than the sole protection. Keep it for early rejection + logging.
